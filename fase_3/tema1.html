¿Qué es HTML5 semántico?

Si sabe un poco sobre HTML, sabrá que las etiquetas HTML se utilizan (principalmente) para formatear el contenido: estas etiquetas le dicen al navegador cómo mostrar el contenido en la página. No dan ninguna indicación sobre qué tipo de contenido contienen o qué papel juega ese contenido en la página.

HTML5 semántico aborda esta deficiencia definiendo etiquetas específicas para indicar claramente qué papel desempeña el contenido que contienen esas etiquetas. Esa información explícita ayuda a los robots / rastreadores como Google y Bing a comprender mejor qué contenido es importante, cuál es una subsidiaria, cuál es para la navegación, etc.

Al agregar etiquetas HTML semánticas a sus páginas, proporciona información adicional que ayuda a Google y Bing a comprender los roles y la importancia relativa de las diferentes partes de su página.

¿Por qué necesito usar etiquetas semánticas HTML5?

Para los usuarios videntes, cuando una página está bien diseñada visualmente, es fácil identificar las diversas partes de una página web de un vistazo. Los encabezados, los menús y (con suerte) el contenido principal son inmediatamente evidentes visualmente. Ahora imagina que eres ciego.

Los bots (arañas) de Google y Bing son, si no ciegos, con graves problemas de visión. Para ellos, las pistas visuales son fenomenalmente difíciles de ver y entender, necesitan tu ayuda.

Si puede comunicar con éxito a Google y Bing qué parte de la página es el encabezado, cuál es el pie de página y cuál es para la navegación, se lo agradecerán. Lo más importante es que al decirles cuál es el contenido más importante, les das una instrucción explícita para priorizar ese contenido.

Para los usuarios ciegos o con discapacidad visual y dependen de lectores de pantalla para describir verbalmente lo que hay en una página, el uso adecuado de los elementos semánticos HTML5 permitirá a los lectores de pantalla comunicar con mayor precisión su contenido al hacer que el visual sea audible. Es vital adoptar esta nueva versión de HTML para que pueda hacer que su contenido sea accesible para todos los posibles visitantes del sitio.

Estructura de etiquetas semánticas HTML5

Ejemplos de etiquetas HTML semánticas incluyen "nav, footer y section. Hay muchas más etiquetas semánticas HTML5 que se pueden usar (por ejemplo, blockquote y em), pero en este artículo solo estoy viendo aquellas etiquetas HTML semánticas que necesitará para dividir el contenido de la página en sus partes básicas. Las siguientes etiquetas HTML5 se pueden usar en lugar de las etiquetas div para dividir el contenido de la página en partes identificadas, cada una de las cuales cumple una función específica. Como puedes imaginar, máquinas como Googlebot y Bingbot AMAN esto.

Barnard, J. (2017, September 25). HTML5 semantic tags: What they are and how to use them! Semrush Blog; Semrush. https://www.semrush.com/blog/semantic-html5-guide/

- Elemento de encabezado: el elemento encabezado define el contenido que debe considerarse la información introductoria de una página o sección.
- Elemento Nav: los enlaces del menú de navegación principal se colocarían en una etiqueta nav. Pero los menús de navegación secundaria en otras partes de la página también podrían obtener uno.
- Etiqueta principal: el cuerpo de una página debe ir en la etiqueta principal, no en las barras laterales y la navegación principal. Solo debe haber uno por página.
- Elemento de artículo: el elemento artículo define el contenido autónomo que podría ser independiente de la página o el sitio en el que se encuentre. Por ejemplo, una entrada de blog.
- Elemento de sección: el uso de sección es una forma de agrupar contenido cercano de un tema similar. Una etiqueta de sección difiere de una etiqueta de artículo porque no es necesariamente autónoma.
- Elemento aparte: un elemento aparte define el contenido que es menos importante. A menudo se usa para barras laterales, áreas que agregan información complementaria pero no vital.
- Elemento de pie de página: usaría pie de página en la base de una página o sección. Puede incluir información de contacto y algo de navegación del sitio.

Esta clara delimitación y atribución explícita de roles a cada parte del contenido hace que la página sea mucho más clara y fácil de indexar correctamente para Google y Bing.

Dado que todas estas etiquetas se comportan exactamente igual que las etiquetas div; Simplemente pueden reemplazar divs existentes en la página sin afectar el diseño. En muchos casos, implementar HTML5 semántico puede ser tan fácil como encontrar el par correcto de div y /div y reemplazarlos.
